3. Giao thức giữa background/background.js (nhận) và content/*.js
  chrome.runtime.onMessage.addListener(((msg, sender, sendResponse)) => {}) (chi tiết xem phần handlers)
  msg = {
          type, // thuộc các dạng "LOG", "SOURCE.", "SUB." (có thể bổ sung sau)
          payload
        }
    3.1. type === "LOG"
            payload = { type, text, url, timestamp }
          sendResponse (tổng): { type: 'LOGGED' }

    3.2. type === "SOURCE.ADD"
            payload = { urls }
          sendResponse (tổng):  { type: 'SOURCE.ADDED', payload: [folderData] }
          hoặc                  { type: 'SOURCE.NOT_ADDED', payload: message (string) }

    3.3. type === "SOURCE.GET_ALL"
            payload = undefined
          sendResponse (tổng): { type: 'SOURCE.LIST', payload: getSourceList() }; // xem storage.js

    3.4. type === "SOURCE.REMOVE"
            payload = { savedAt }
          sendResponse (tổng): { type: 'SOURCE.REMOVED', payload: await removeSource(payload.savedAt)}; // xem storage.js

    3.5. type === "SUB.SEARCH"
            payload = { videoId, folderMode }
          sendResponse (tổng): resolveSubtitles(videoId, folderMode)
            nếu folderMode falsy:
              { type: 'SUB.LIST', payload: candidates }
            nếu folderMode truthy:
              { type: 'SOURCE.LIST', payload: await getSourceList() }

    3.4. type === "SUB.SELECT"
            payload = { videoId, candidate }
          sendResponse (tổng): processSubtitles(videoId, candidate, rawText)
              { type: 'SUB.READY', payload: subObj }
    
    3.6. type === "SUB.USE_CACHE"
            payload = { videoId }
          sendResponse (tổng): { type: 'SUB.READY', payload: await useSubData(videoId) }
    
    3.6. type === "SUB.GET_ALL"
            payload = { videoId }
          sendResponse (tổng): { type: 'SUB.LIST', payload: await getSubDataList(videoId) }
    
    3.7. type === "SUB.REMOVE"
            payload = { videoId }
          sendResponse (tổng): { type: 'SUB.REMOVED', payload: await removeSubData(videoId) };

    3.8. type === "SUB.LOCAL"
            payload = { videoId, rawText, fileName }
          sendResponse (tổng): processSubtitles(videoId, localFileData (xử lí từ fileName), rawText)

    3.9. type === "RENDER" (UI gửi yêu cầu render thủ công cho background)
            payload = { videoId }
          sendResponse (tổng): { type: 'RENDER.REQUEST_RESULT', payload: boolean }

    3.10. Dữ liệu khác chuẩn/chạy handlers bị lỗi
          sendResponse (tổng): { type: 'ERROR', payload: err.message hoặc 'Unknown action' }

Cơ chế các tag trong Aegisub
1.  Định nghĩa khung chữ: khung hình chữ nhật cùng phương, chiều với chữ (ở góc xoay 0 độ), 
    bao hàm toàn bộ chữ trong nội dung của dòng (line.text).
    - Kích thước-tọa độ khung chữ không phụ thuộc màu sắc, thuộc tính của viền và bóng của chữ

2.  Phân loại thuộc tính (tag) của chữ
  2.1.  (Clip) Thuộc tính toàn cục, độc lập: áp dụng 1 lần mỗi dòng, không thay đổi kích thước khung chữ
    \clip hoặc \iclip (cái cuối cùng có hiệu lực)
      * Chú ý: nếu có nhiều \clip lẫn \iclip (trong \t), thì \?clip cuối cùng quyết định loại clip (\clip hoặc \iclip) cho toàn bộ các tag \?clip trước đó.

  2.2.  (Collision) Thuộc tính toàn cục, can thiệp: áp dụng 1 lần mỗi dòng, có thay đổi dữ liệu khung chữ
    * \an: thay đổi tọa độ khung chữ (cái đầu tiên có hiệu lực)
    \org: vô hiệu hóa collision (cái đầu tiên có hiệu lực)
    \pos hoặc \move: vừa thay đổi tọa độ khung vừa vô hiệu collision (cái đầu tiên có hiệu lực)
    \t: vô hiệu hóa collision

  2.3.  (Decoration Local Tags) Thuộc tính cục bộ, độc lập: áp dụng mỗi cụm từ (chia cụm từ thành các phần riêng biệt), không thay đổi kích thước khung chữ
    Các tag liên quan đến màu sắc, viền, bóng của chữ, \fa<x-y>,
    * \fr<x-y-z>, 

  2.4.  (Layout Local Tags) Thuộc tính cục bộ, can thiệp: áp dụng mỗi cụm từ, có thay đổi kích thước khung chữ
    Các tag liên quan đến kích cỡ chữ (\fs, \fsc), dấu cách và xuống dòng (breaks như \h, \N, \n),
    các tag thay đổi font (\b, \i, \fn, \r),
    * \fsp
