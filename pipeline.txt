3. Giao thức giữa background/background.js (nhận) và content/*.js
  chrome.runtime.onMessage.addListener(((msg, sender, sendResponse)) => {}) (chi tiết xem phần handlers)
  msg = {
          type, // thuộc các dạng "LOG", "SOURCE.", "SUB." (có thể bổ sung sau)
          payload
        }
    3.1. type === "LOG"
            payload = { type, text, url, timestamp }
          sendResponse (tổng): { type: 'LOGGED' }

    3.2. type === "SOURCE.ADD"
            payload = { urls }
          sendResponse (tổng):  { type: 'SOURCE.ADDED', payload: [folderData] }
          hoặc                  { type: 'SOURCE.NOT_ADDED', payload: message (string) }

    3.3. type === "SOURCE.GET_ALL"
            payload = undefined
          sendResponse (tổng): { type: 'SOURCE.LIST', payload: getSourceList() }; // xem storage.js

    3.4. type === "SOURCE.REMOVE"
            payload = { savedAt }
          sendResponse (tổng): { type: 'SOURCE.REMOVED', payload: await removeSource(payload.savedAt)}; // xem storage.js

    3.5. type === "SUB.SEARCH"
            payload = { videoId, folderMode }
          sendResponse (tổng): resolveSubtitles(videoId, folderMode)
            nếu folderMode falsy:
              { type: 'SUB.LIST', payload: candidates }
            nếu folderMode truthy:
              { type: 'SOURCE.LIST', payload: await getSourceList() }

    3.4. type === "SUB.SELECT"
            payload = { videoId, candidate }
          sendResponse (tổng): processSubtitles(videoId, candidate, rawText)
              { type: 'SUB.READY', payload: subObj }
    
    3.6. type === "SUB.USE_CACHE"
            payload = { videoId }
          sendResponse (tổng): { type: 'SUB.READY', payload: await useSubData(videoId) }
    
    3.6. type === "SUB.GET_ALL"
            payload = { videoId }
          sendResponse (tổng): { type: 'SUB.LIST', payload: await getSubDataList(videoId) }
    
    3.7. type === "SUB.REMOVE"
            payload = { videoId }
          sendResponse (tổng): { type: 'SUB.REMOVED', payload: await removeSubData(videoId) };

    3.8. type === "SUB.LOCAL"
            payload = { videoId, rawText, fileName }
          sendResponse (tổng): processSubtitles(videoId, localFileData (xử lí từ fileName), rawText)

    3.9. type === "RENDER" (UI gửi yêu cầu render thủ công cho background)
            payload = { videoId }
          sendResponse (tổng): { type: 'RENDER.REQUEST_RESULT', payload: boolean }

    3.10. Dữ liệu khác chuẩn/chạy handlers bị lỗi
          sendResponse (tổng): { type: 'ERROR', payload: err.message hoặc 'Unknown action' }

6. Cấu trúc code
  6.1. background.js (ko tính các hàm import)
    1. renderSendData(): Hàm thực thi gửi dữ liệu cho renderer (thực ra là cho cả content-side nhưng chỉ renderer nhận)
    2. onClickedListener(): Hàm lập trình xử lí của background khi nhấn vào icon extension (chạy luôn)
    3. onHandlersListener(): Hàm lập trình giao tiếp của background-side với content-side (chạy luôn)
    - Phần định nghĩa giao thức (cho onHandlersListener() ở trên)
    4. resolveSubtitles(): Hàm xử lý và điều phối tìm kiếm phụ đề dựa trên videoId 
      (ko tìm bố... à nhầm, cache, vì đã có hàm của storage tìm hộ)
    5. processSubtitles(): Hàm xử lý và lưu cache.

  6.2. ui.js
    1. Các hàm bổ trợ cơ bản
      1.1. sendLogToBackground(): gửi log về background.js (content-side về background-side)
      1.2. toggleOverlay(): hàm điều khiển ẩn/hiện UI
      1.3. getRelativeTimeString(): tính thời gian (N (đơn vị) trước) và tuyệt đối (HH:MM:SS DD/MM/YYYY)
      1.4. updateVideoIdInUI(): Hàm cập nhật video ID vào UI (hiện tại chỉ hỗ trợ YT.)
    - Đặt obj uiData để quản lí dữ liệu toàn file
    2. Các hàm hạng mục tính năng
      2.1. buildMainHTML(): Hạng mục 1. Khởi tạo khung UI và API của nó.
      2.2. buildTabListLogic(): Hạng mục 2. Khởi tạo logic trên danh sách trang hiển thị.
        2.2.1. selectTab(): xử lí lựa chọn trang
      2.3. buildDragFeature(): Hạng mục 3. Tính năng di chuyển giao diện.
        2.3.1. handleMove(): xử lí tọa độ UI khi di chuyển
        2.3.2. barTitleOnClick(): chuyển trạng thái UI sang kéo thả (chuyển vị trí), khi bấm vào thanh tiêu đề
        2.3.3. barTitleOnClickHold(): xử lí hành động giữ nhấn + di chuyển chuột
        2.3.4. barTitleOnTouchHold(): xử lí nhấn giữ + di chuyển chạm
        2.3.5. barTitleOnRelease(): xử lí thả chuột/chạm
      2.4. buildSourceManagerTab(): Hạng mục 4. Tính năng trong tab 1: Quản lí nguồn.
        2.4.1. renderLinkList(): render danh sách nguồn
        2.4.2. initSourceList(): hàm thực hiện tải danh sách nguồn từ background.js và chạy render danh sách nguồn.
      2.5. buildSubtitleManagerTab(): Hạng mục 5. Tính năng trong tab 2: Quản lý phụ đề.
        2.5.1. renderSubFileArray(): render danh sách tệp phụ đề
        2.5.2. initSubFileArray(): hàm thực hiện tải danh sách tệp phụ đề từ background.js và chạy render danh sách tệp phụ đề.
        - Cụ thể: nhận searchId (đoạn Id mà người dùng nhập), targetId (Id trên tab đang hiển thị), cacheSearchMode chế độ quét cache/online (true = cache, false = online)
          + targetId sẽ truyền trực tiếp cho renderSubFileArray()
          + Xử lí searchId và cacheSearchMode: Nếu searchId trống thì tìm toàn bộ. Nếu searchId ko trống thì tìm theo Id (trùng)
            - nếu tìm toàn bộ trong cache: thì SUB.GET_ALL với {videoId: searchId} 
            - nếu tìm toàn bộ trong các nguồn online: thì SUB.SEARCH với {videoId: searchId, folderMode = false}
    - Phần chạy chính (chạy các hàm Hạng mục và kiểm tra): 
      2.1 > 2.2 > 2.3 > 2.4 > 2.5.
  
  6.3. renderer.js
    1. Các hàm bổ trợ cơ bản
      1.1. sendLogToBackground(): gửi log về background.js (content-side về background-side)
      1.2. updateVideoIdInRenderer(): Hàm cập nhật video ID vào UI (hiện tại chỉ hỗ trợ YT.)
      1.3. selectVideo(): Hàm chạy tự động tìm, chọn video, và tạo lập khung phụ đề ban đầu.
      1.4. renderDataValidate(): Hàm kiểm tra và hợp lệ hóa dữ liệu (chỉ chạy 1 lần khi chạy hàm chính render())
      - Phần khai báo dữ liệu chung.
    2. Các hàm tính năng bám bắt video (hàm chính: 2.3)
      2.1. computeAspectFitBounds(): Hàm tính toán khung phụ đề (dựa trên tỉ lệ khung hình cố định của video và của sub)
      2.2. applyOverlayBounds(): Hàm áp dụng tọa độ-kích thước mới tính toán vào khung phụ đề hiện có
      2.3. observeParentLayout(): Hàm thiết lập trình bám bắt video
    3. Các hàm tính năng render
      3.1. styleObjToCss(): Hàm chuyển đổi style từ object sang bộ CSS tĩnh (cấu trúc 2 lớp vỏ-ruột)
      3.2. scaler(): Hàm thực thi chuyển đổi style theo thay đổi kích thước khung hình (từ khung PlayResX-Y sang khung width-height của phụ đề)
      3.3. pendingStylesCalculate(): Hàm tính toán và áp dụng style mới vào hàng chờ
      3.4. applyPendingStyles(): Hàm áp dụng style đang chờ vào trước khi frame loop render tiếp.
      3.5. renderSubtitleFrame(): Hàm xử lí dữ liệu render theo từng frame
      3.6. clearSubtitleFrame(): Hàm xóa dữ liệu render trong frame
      3.7. enableRenderLoop(): Hàm mở chạy vòng lặp render theo frame
      3.8. disableRenderLoop(): Hàm hủy chạy vòng lặp render theo frame
      3.9. render(): Hàm chạy render chính
    - Phần trình xử lí phản hồi, tự động lấy cache và render
